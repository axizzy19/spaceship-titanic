## Baseline

Первичный вывод: \
- Можно объединить признаки RoomService, FoodCourt, ShoppingMall, Spa, VRDeck в один признак. \
- Необходимо обработать пропуски. \
- True/False нужно преобразовать в 0/1 для корреляции. \
- От PassengerId и Name целевой признак, скорее всего, не зависит. \
- Необходимо как-то преобразовать тип кают. \

Что было сделано: \
1) объединены признаки в TotalSpent
2) сделана матрица корреляции. Из матрицы корреляций видно, что больше всего влияние имеет CryoSleep, а отрицательную корреляцию дает признак HasSpent.
3) проверена важность признаков, входящих в TotalSpent. Все признаки оказались важны, их удалять нельзя
4) Преобразован тип кают. Выявлено, что наибольгий шанс к транспортировке имеют палубы B, C
5) Проверена зависимость Transported от HomePlanet. По результатам выведенных графиков можно сделать вывод, что то, откуда прибыл пассажир является важным для прогнозирования модели
6) Далее проведено обучение моделей, их сравнение. На наилучших моделях сделаны ансамбли. Лучший рещультат показал ансамбль Stacking
7) Далее была попытка улучшить модель с помощью признаков, но результат ухудшился
8) Таким образом, наилуший результат - 0,75

Вывод: \
Для улучшения модели была проведена попытка преобразования признаков, но это не дало улучшения. Также не дало улучшение применение других ансамблей...
